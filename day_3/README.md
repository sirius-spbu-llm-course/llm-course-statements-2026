# Задание по работе с проектом

## Инструмент для стеганографии в изображениях
Инструмент для скрытия текстовых сообщений в изображениях методом LSB (Least Significant Bit).

Программа встраивает данные в младшие разряды значений пикселей, что делает изменения визуально незаметными.

## Требования

- Компилятор C++ с поддержкой C++17
- Make (для Linux/macOS)

## Сборка (Linux / macOS)

Клонируйте репозиторий и скачайте зависимости:

```bash
git clone <repository-url>
cd steganography
```

Уберите в Makefile из команды сборки флаг -static (16 строчка)

```
$(EXE): $(BINDIR) $(OBJECTS)
	$(CXX) $(OBJECTS) -o $(EXE) [-static]
```

Соберите проект:

```
make
```

## Сборка (Windows)

### Вариант 1: Через CLion

1. Создайте новый проект через VCS вписав http репозитория, либо открыв скаченный

2. Если при этом всплыло окно про выбор таргета для сборки - впишите `all` или `stego`

3. Всё должно само собраться. При этом запускать полученный бинарный файл можно или из консоли с аргументами, или прописать в настройки запуска CLion Run with arguments нужные аргументы

### Вариант 2: MinGW-w64

Установите MinGW-w64 (например, через https://winlibs.com или MSYS2).

Откройте командную строку в папке проекта и выполните:

```bash
g++ -std=c++17 -O2 -Iinclude -o stego.exe src/main.cpp src/args_parser.cpp src/image.cpp src/encoder.cpp src/decoder.cpp
```

или отыщите mingw32-make, пропишите его в PATH и выполните

```
mingw32-make.exe all
```

## Использование

Кодирование сообщения:

```bash
./stego encode -i input.png -o output.png -m "Secret message"
./stego encode -i input.png -o output.png -m "Hello" --seed 12345 --base 8
```

Декодирование сообщения:

```bash
./stego decode -i output.png
./stego decode -i output.png --seed 12345 --base 8
```

## Задания
Текущая реализация работает, но имеет ограничения. Необходимо расширить функциональность.

### Задание 1: Использование всех каналов

Сейчас данные записываются только в красный канал (R). Реализуйте использование всех трех каналов RGB, что увеличит емкость в 3 раза.

### Задание 2: Случайный порядок пикселей

Сейчас параметр --seed принимается, но не используется. Реализуйте перемешивание порядка пикселей на основе seed, чтобы без знания seed нельзя было извлечь сообщение.
